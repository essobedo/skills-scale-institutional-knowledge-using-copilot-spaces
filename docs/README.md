# OctoAcme Project Management Docs

This README provides a central index of OctoAcme project management process documents and a high-level summary of how projects are run.

## Process Summary

OctoAcme uses a lightweight, iterative project management approach across five lifecycle stages: initiation, planning, execution, release, and retrospective improvement. Work is guided by clear ownership (PM and Product Lead), measurable outcomes, risk tracking, regular communication cadences, and continuous improvement after milestones and incidents.

### Lifecycle Stages

1. **Initiation** — Validate the business need, align stakeholders, define success criteria, and create a lightweight project one-pager before committing to full planning.
2. **Planning** — Break work into shippable increments, identify dependencies and risks, align timelines, and define the Definition of Done.
3. **Execution & Tracking** — Run daily standups, use a project board to track progress, maintain quality through testing and code review, and escalate blockers promptly.
4. **Risk Management & Communication** — Maintain a Risk Register, follow regular communication cadences (weekly PM/PdM sync, twice-weekly standups, monthly stakeholder updates), and manage escalations.
5. **Release & Deployment** — Follow a structured release checklist, validate in staging, coordinate deployment windows, and communicate the release to stakeholders.
6. **Retrospective & Continuous Improvement** — After each milestone or incident, capture learnings, celebrate wins, assign action items, and track improvements over time.

### Core Principles

- **Customer-first**: prioritize customer value and usability.
- **Iterative delivery**: ship small, testable increments.
- **Clear ownership**: every project has a named PM and Product Lead.
- **Data-informed decisions**: measure impact and iterate based on evidence.
- **Psychological safety**: encourage open feedback and continuous learning.
